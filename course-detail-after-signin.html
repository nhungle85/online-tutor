Instruction

  • Functions in JavaScript are pieces of code you can write and reuse over and over again.
  • Other programming languages may call them subroutines or procedures, but the concept is the same.
  • There are multiple ways to create functions. In JavaScript, the function declaration is the oldest way.
  • To write a function declaration, you first use the function keyword, followed by a function name unique in the scope. You will need this name to call the function later.
  • You will be using the JS section of your Codepen, and you will need to open the Codepen Console as well (button in lower left)
  • Be aware that the Codepen Console does not log error messages. If you suspect that you have an error in your code, open the browser developer console to view error messages.

Instruction

  • Function expressions are a way to write functions inline in expressions. This can be useful for various reasons, including being able to use a function as an argument for another function (which is known as a callback function).
  • The function expression was added to JavaScript after function declarations.
  • In the above example, we created a variable named adder, then we assigned a function as its value, which we created using the function keyword, a parameter list, and a return statement inside a set of curly braces -- which looks just like a function declaration, except that there is no name after the function keyword.
  • There are actually two variants of the function expression: the anonymous function expression, and the named function expression. Above, you see an example of an anonymous function expression.
  • However, if you tried to call this function using someName(1,2), it would not work. The function name someName is only valid inside its own function body and does not exist outside of it. Most function expressions you run into will be anonymous. There are cases when a named function expression is useful, but we will not discuss them at this time.

Instruction

  • Arrow functions are a new introduction to JavaScript in ES6 (ES2015).
  • Arrow functions will be used very frequently in React (and other modern libraries), so it is crucial that you learn to understand and write them.
  • Arrow functions are a more compact version of function expressions and are frequently used as callback functions (functions that are passed into other functions as arguments). You will learn more about using arrow functions as callback functions later.
  • Aside from being more compact, the other crucial difference with arrow functions is that they do not have their own this keyword binding. We will not delve into the meaning of this at the moment, but you will learn more later.
